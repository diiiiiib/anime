إعداد قواعد أمان Firebase

مهم: يجب تعديل قواعد الأمان في Firestore Database للسماح للمستخدمين المسجلين بالقراءة والكتابة.

الخطوات المطلوبة:

الخطوة 1: فتح Firebase Console

افتح Firebase Console

الخطوة 2: اختيار المشروع

اختر مشروعك (xodiib) من القائمة

الخطوة 3: فتح Firestore Database

من القائمة الجانبية، اختر Firestore Database

الخطوة 4: إنشاء قاعدة البيانات (إذا لم تكن موجودة)

إذا لم تكن قاعدة البيانات مفعلة، اضغط على Create Database

اختر Start in test mode (للتجربة)

اختر منطقة قريبة منك (مثل: europe-west1)

الخطوة 5: تعديل قواعد الأمان (Rules)

في صفحة Firestore Database، اضغط على تبويب Rules

استبدل القواعد الحالية بالقواعد التالية:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // السماح للمستخدمين المسجلين بالقراءة والكتابة
    match /users/{userId} {
      allow read, write: if request.auth != null;
    }

    match /conversations/{conversationId} {
      allow read, write: if request.auth != null;
    }

    match /conversations/{conversationId}/messages/{messageId} {
      allow read, write: if request.auth != null;
    }
  }
}

اضغط على Publish لحفظ التغييرات

ملاحظة: بعد تعديل القواعد، قد يستغرق الأمر بضع دقائق حتى تصبح التغييرات سارية المفعول.

استكشاف الأخطاء

الخطأ: "Missing or insufficient permissions"

السبب: قواعد الأمان لا تسمح بالوصول

الحل: اتبع الخطوات أعلاه لتعديل القواعد

الخطأ: "User not found"

السبب: المستخدم غير مسجل

الحل: تأكد من تسجيل الدخول بشكل صحيح

الخطأ: "Network request failed"

السبب: مشكلة في الاتصال بالإنترنت

الحل: تحقق من اتصالك بالإنترنت
